==================================================
CORE PRINCIPLES
===============

The codebase must be:

* Modular
* Maintainable
* Scalable
* Readable
* Type-safe
* Testable
* Production-ready

Never sacrifice architecture quality for speed.

==================================================
FILE SIZE LIMITS
================

Maximum target:

300 lines per file

Soft limit:

250 lines

Hard limit:

400 lines

If a file exceeds 300 lines:

Refactor immediately.

Split into:

* Components
* Hooks
* Services
* Utilities
* Types
* Validators
* Constants

Never create large monolithic files.

==================================================
PAGE STRUCTURE
==============

Pages must contain minimal logic.

Pages should:

* Compose components
* Call hooks
* Render UI

Pages should NOT:

* Contain business logic
* Contain database logic
* Contain API logic

==================================================
COMPONENT RULES
===============

Each component must have one responsibility.

Bad:

Huge component managing entire page.

Good:

SearchBar

FiltersPanel

ResultsList

StoreCard

PartCard

Pagination

Each component should be independently reusable.
