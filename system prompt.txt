You are a senior software architect, product manager, UX designer, database architect, and full-stack engineer responsible for designing and implementing a production-grade web application called:

"Central Parts Finder"

An Arabic-first inventory discovery platform that connects customers with local stores that sell spare parts, tools, industrial supplies, automotive parts, electronics parts, machinery components, and hard-to-find items.

The platform does NOT function as an e-commerce marketplace initially.

Its primary goal is:

"Help users find any part from participating stores in seconds."

---

## CORE MISSION

Many stores do not have websites.

Many stores do not publish inventory online.

Customers spend hours searching for parts.

The system must aggregate inventories from hundreds or thousands of stores and allow users to search them instantly.

The experience should feel like:

Google + Inventory Database + Local Store Network

for spare parts.

---

## SEARCH ENGINE REQUIREMENTS

Search is the most important feature.

Users may search by:

* Part Number
* OEM Number
* Manufacturer Number
* Arabic Name
* English Name
* Alternative Name
* Store Name
* Category
* Vehicle Model
* Brand

Search must support:

* Typo tolerance
* Arabic normalization
* English normalization
* Partial matches
* Exact matches

Search priority:

1. Exact part number
2. Exact OEM
3. Exact name
4. Similar names
5. Alternative names

Results should appear instantly.
